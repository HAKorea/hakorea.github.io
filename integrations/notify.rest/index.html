REST 알림

rest 알림 플랫폼을 사용하면 RESTful 알림을 홈어시스턴트에서 다른 상대방에게 전달할 수 있습니다.

설치시 REST 알림을 활성화하려면 configuration.yaml 파일에 다음을 추가하십시오.

# Example configuration.yaml entry
notify:
  - name: NOTIFIER_NAME
    platform: rest
    resource: http://IP_ADDRESS/ENDPOINT

Configuration Variables

name

(string)(Optional)

notify.NOTIFIER_NAME. 선택적 매개 변수name을 설정하면 여러 알리미를 만들 수 있습니다. 알리미는 서비스 notify.NOTIFIER_NAME 에 바인딩합니다.

Default value:

notify

resource

(string)(Required)

값을 받을 자원 또는 엔드 포인트.

method

(string)(Optional)

요청 방법. 유효한 옵션은 GET, POST 혹은 POST_JSON.

Default value:

GET

verify_ssl

(boolean)(Optional)

엔드 포인트의 SSL 인증서를 확인.

Default value:

true

authentication

(string)(Optional)

HTTP 인증의 유형 basic 혹은 digest.

Default value:

basic

username

(string)(Optional)

REST 엔드 포인트에 액세스하기위한 사용자 이름

password

(string)(Optional)

REST 엔드 포인트에 액세스하기위한 비밀번호

headers

(string)(Optional)

요청에 대한 헤더.

message_param_name

(string)(Optional)

메시지의 매개 변수 이름.

Default value:

message

title_param_name

(string)(Optional)

제목의 매개 변수 이름.

target_param_name

(string)(Optional)

대상의 매개 변수 이름.

data

(string)(Optional)

자원으로 보낼 추가 매개 변수 사전(dictionary).

data_template

(template)(Optional)

자원으로 보낼 추가 매개 변수의 템플리트 사전.

알림을 사용하려면 getting started with automation page 를 참조하십시오 .
