Custom Panel

The panel_custom support allows you to add additional panels to your Home Assistant frontend. The panels are listed in the sidebar if wished and can be highly customized. See the developer documentation on instructions how to build your own panels.

To enable customized panels in your installation, add the following to your configuration.yaml file:

# Example configuration.yaml entry
panel_custom:
  - name: my-panel
    sidebar_title: TodoMVC
    sidebar_icon: mdi:work
    url_path: my-todomvc
    js_url: /local/my-panel.js
    config:
      who: world

Store your custom panels in <config>/www to make them available in the frontend at the path /local.

Configuration Variables

name

(string)(Required)

Name of the web integration that renders your panel.

sidebar_title

(string)(Optional)

Friendly title for the panel in the sidebar. Omitting it means no sidebar entry (but still accessible through the URL).

sidebar_icon

(icon)(Optional)

Icon for entry. Pick an icon that you can find on materialdesignicons.com to use for your input and prefix the name with mdi:. For example mdi:car, mdi:ambulance, or mdi:motorbike.

Default value:

mdi:bookmark

url_path

(string)(Optional)

The URL your panel will be available on in the frontend. If omitted will default to the panel name.

js_url

(string)(Exclusive)

The URL that contains the JavaScript of your panel. This is exclusive to module_url and webcomponent_path.

module_url

(string)(Exclusive)

The URL that contains the JavaScript module of your panel. Loaded as a JavaScript module instead of a script. This is exclusive to js_url and webcomponent_path.

config

(list)(Optional)

Configuration to be passed into your web component when being instantiated.

require_admin

(boolean)(Optional)

If admin access is required to see this panel.

Default value:

false

embed_iframe

(boolean)(Optional)

Set to true to embed panel in iframe. This is necessary if the panel is using the React framework or if it contains conflicting web components.

Default value:

false

trust_external_script

(boolean)(Optional)

By default the user has to confirm before loading a script from an external source. Setting this to true will omit this confirmation.

Default value:

false

webcomponent_path

(string)(Exclusive)

DEPRECATED The HTML path to your component. If omitted will default to <config dir>/panels/<component name>.html This is exclusive to js_url and module_url.
