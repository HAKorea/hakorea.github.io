리만합(Integration - Riemann sum)

integration 플랫폼은 소스 센서가 제공한 값의 Riemann sum을 제공합니다. Riemann sum은 finite sum에 의한 integral의 근사치입니다. source가 변경되면 integration 센서가 업데이트됩니다. 빠른 샘플링 소스 센서로 더 나은 결과를 제공합니다. 이 구현에서 기본값은 사다리꼴(Trapezoidal) 방법이지만 왼쪽과 오른쪽(Left and Righ) 방법을 선택적으로 사용할 수 있습니다.

설정

설치시 Integration 센서를 활성화하려면 configuration.yaml 파일에 다음을 추가하십시오.

# Example configuration.yaml entry
sensor:
  - platform: integration
    source: sensor.current_power

Configuration Variables

source

(string)(Required)

The entity ID of the sensor providing numeric readings.

name

(string)(Optional)

Name to use in the frontend.

Default value:

source entity ID meter

round

(integer)(Optional)

Round the calculated integration value to at most N decimal places.

Default value:

3

unit_prefix

(string)(Optional)

Metric unit to prefix the integration result. Available units are k, M, G and T.

Default value:

None

unit_time

(string)(Optional)

SI unit of time to integrate over. Available units are s, min, h and d.

Default value:

h

unit

(string)(Optional)

Unit of measurement to be used for the integration.

method

(string)(Optional)

Riemann sum method to be used. Available methods are trapezoidal, left and right.”

Default value:

trapezoidal

온/오프 전기 보일러와 같이 소비량이 많은 기기가 있는 경우 정확한 판독값을 얻으려면 left 방법을 선택해야합니다. unit이 설정되면 unit_prefix와 unit_time은 무시됩니다.

Energy

integration 센서는 에너지가 일반적으로 kWh로 청구되고 많은 센서가 W(Watts)로 전력을 공급하기 때문에 에너지 청구 시나리오에서 매우 유용합니다.

Watts 단위의 전력 측정값을 제공하는 센서가 있는 경우 (W를 unit_of_measurement로 사용), integration 센서를 사용하여 소비되는 에너지량을 추적할 수 있습니다. 다음 설정을 예로 들어 보겠습니다.

sensor:
  - platform: integration
    source: sensor.current_power
    name: energy_spent
    unit_prefix: k
    round: 2

이 설정은 에너지를 kWh 단위로 sensor.energy_spent를 제공합니다.
