기본 정보 설정

기본적으로 처음시작시 일부 기능으로, Home Assistant는 IP 주소 지리적 위치에서 사용자의 위치를 ​​감지할 수 있습니다. 홈어시스턴트는 이 위치를 기준으로 온도 단위, 시간대를 자동으로 선택합니다. 처음 시작할 때 혹은 나중에 설정 -> 일반에 들어가면 조정할 수 있습니다.

YAML을 선호하는 경우 configuration.yaml 파일에 다음 정보를 추가할 수 있습니다 :

homeassistant:
  latitude: 32.87336
  longitude: 117.22743
  elevation: 430
  unit_system: metric
  time_zone: America/Los_Angeles
  name: Home
  whitelist_external_dirs:
    - /usr/var/dumping-ground
    - /tmp

Configuration Variables

latitude

(float)(Optional)

해가 뜨고지는 시간을 계산하는데 필요한 위치의 위도.

longitude

(float)(Optional)

해가 뜨고지는 시간을 계산하는데 필요한 위치의 경도.

elevation

(integer)(Optional)

해발 고도(미터) 날씨/일출 데이터에 영향을 줍니다.

unit_system

(string)(Optional)

metric 미터법, imperial 영국식.

time_zone

(string)(Optional)

컬럼에서 당신의 TZ을 Wikipedia’s list of tz database time zones 중에서 선택

name

(string)(Optional)

홈어시스턴트가 실행중인 위치의 이름.

customize

(string)(Optional)

Customize entity들.

customize_domain

(string)(Optional)

Customize domain에 모든 entity.

customize_glob

(string)(Optional)

Customize 패턴이 매칭되는 모든 entities.

whitelist_external_dirs

(list)(Optional)

파일을 보내기 위한 소스로 사용할 수 있는 폴더 목록

Core Service 리로드

홈어시스턴트는 homeassistant.reload_core_config 서비스가 실행되는 동안 Core 설정을 다시 로드하는 서비스를 제공합니다. 이를 통해 위의 섹션을 변경하고 홈어시스턴트를 다시 시작할 필요없이 적용되는 것을 확인할 수 있습니다. 이 서비스를 호출하려면 개발자 도구 아래의 “서비스”탭으로 이동하여 homeassistant.reload_core_config 서비스를 선택 하고 “서비스 호출”버튼을 클릭하십시오. 또는 설정 > 서버 제어 > 일반에서 “위치와 사용자 정의 다시로드” 버튼을 누를 수 있습니다.
